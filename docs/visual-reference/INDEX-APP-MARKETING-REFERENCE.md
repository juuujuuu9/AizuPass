# Index.app — marketing page structure & storytelling (notes)

> Compiled: 2026-03-27  
> Source: [index.app](https://index.app/) (referrer in original link: `?ref=minimal.gallery`)

## Summary

[Index](https://index.app/) sells an **analytics platform with a high-touch onboarding story**: hero and social proof → **three steps that emphasize “we do the work”** → **AI unlock** after setup → **benefit grid** → **deep feature pillars** (connect, AI, dashboards, sharing) → **secondary capabilities** (prompts, follow-ups, favorites, alerts) → **FAQs** → **repeated closing CTA** → footer. The narrative is **B2B / decision-maker** oriented: speed to insight, less reporting labor, unified data, revenue impact.

## Macro flow (top to bottom)

| Stage | What the page does |
|-------|---------------------|
| Nav + urgency | Sticky nav: Features, Blog, Updates, Pricing, Careers, Login, **Book demo**. Banner: **INDEX 2.0 EARLY PREVIEW** (momentum / FOMO). |
| Hero | Headline: **“Analytics done for you.”** Sub: grow faster, all-in-one analytics. Primary CTA: **Book demo** (sales-led, not “start free” only). |
| Credibility | **“Trusted by some of the biggest companies”** + logo row (enterprise signal early). |
| Process | **“Get started in 3 simple steps”** — Onboard call → Add connections → **Sit back and relax / While we build things.** Positions the product as **partner-led setup**, not purely self-serve. |
| Bridge | **“With setup done, unlock accurate AI-driven analytics.”** Frames AI as the **reward** after onboarding. |
| Benefit grid (4) | Fast + accurate answers · Less reporting work · Unified data · Drive revenue with data — **outcomes before screens**. |
| Product pillars | **“What you get with an end-to-end analytics platform”** then long-form blocks: **Connect** (sources, naming, no CSV shuffle) · **AI** (plain English → SQL + chart, guardrails) · **Dashboards** · **Sharing** (Slack, email, access control). Each: headline + short italic positioning line + bullets. |
| Feature chips | Short sections: Suggested prompts, Follow-ups, Favorite threads, Real-time alerts — **breadth** without another full pillar layout. |
| FAQs | Accordion: SQL, sources, security, setup time, freshness. |
| Closing band(s) | **“Insights without the wait”** + sub + **Book demo** (repeated pattern — often used for sticky/mobile conversion). |
| Footer | Product / Company / Legal; Careers **HIRING** callout. |

## Storytelling patterns (what to steal)

1. **“Done for you” in the hero** — Matches the three-step story (call → connections → they build). Reduces fear of a heavy analytics project.

2. **Demo-first CTA** — **Book demo** everywhere signals **complex sale**, trust-building, and implementation talk — different from PLG “Sign up” but appropriate for data products.

3. **Logos after hero, before depth** — B2B buyers look for **peer proof** before investing attention in feature scroll.

4. **Setup → unlock** — Same narrative device as “onboarding complete” in apps: **AI-driven analytics** is what you *unlock*, not the first screen you see.

5. **Two-layer feature storytelling** — A **tight 2×2 (or 4-up) outcome grid**, then **pillar sections** with product detail. Skimmers get outcomes; evaluators get SQL, Slack, guardrails, etc.

6. **Italic sub-lines under H2s** — e.g. *Index plugs into your data…* / *IndexAI turns natural-language questions…* — quick **product voice** without cluttering bullets.

7. **FAQ as procurement** — Questions map to **security, time-to-value, and ongoing freshness** — what committees ask.

8. **Version banner** — **2.0 early preview** creates **recency** and invites return visitors to re-evaluate.

## Contrast with the block-colored “chapter” landing (printspace-style)

| Dimension | Index-style (this page) | Chapter-block / lifestyle landing |
|-----------|-------------------------|-----------------------------------|
| Primary CTA | Book demo (sales) | Often “Get started” / trial |
| Proof | Logo bar, FAQ, compliance-ish answers | Photography, craft, emotion |
| Process | **We** build after you connect | User uploads / sells |
| Middle | Outcome grid + long feature pillars | Alternating color bands + one idea per band |

Both use **clear vertical order** and **progressive disclosure**; Index leans **rational and enterprise**, the other leans **sensory and creator**.

## Adaptation ideas for QR Check-In

- If you ever offer **white-glove onboarding** or **done-for-you import**, the **three-step “call → connect → we build”** arc is a template.
- **“Unlock X after setup”** maps to: after CSV or integration is live, **unlock** offline queue, real-time door stats, or exports — honest phrasing per [MASTER-PLAN.md](../MASTER-PLAN.md).
- **Outcome grid before screenshots** can mirror: door speed, accuracy, offline, ops — then pillar blocks for staff app, admin, integrations.
